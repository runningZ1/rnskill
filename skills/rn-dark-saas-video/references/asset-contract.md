# Asset Contract

## Preferred Inputs

- product name
- product logo or mark
- one-line promise
- 3-5 pain words or category words
- one example prompt
- one CTA label
- 3-6 output examples
- 4-10 integration/model/provider names
- 3-6 export formats or delivery artifacts
- product screenshots, if available

## Fallbacks

If no screenshots exist, use abstract product-like UI:

- prompt card
- app canvas
- template cards
- provider pills
- model badges
- export folder and pills

If no logo exists, use a simple geometric mark, but label it as placeholder in
the production note.

If integration logos are unavailable, use text pills instead of fake logos.

## Product Accuracy

Do not invent product capabilities silently. If a scene module is stylistically
useful but the capability is unknown, phrase it generically or ask one question.

Examples:

- Unknown local support -> "Works where you do" instead of "Runs locally"
- Unknown export formats -> "Export-ready" instead of "PDF / PPTX / API"
- Unknown model support -> "Bring your stack" instead of naming models
