# Anti-PPT Gate

Use this checklist before implementation and before final delivery.

## Plan-Level Gate

Reject or revise the plan if any answer is "no":

- Does the video have one continuous visual system?
- Does each beat have a main moving object?
- Does each beat show a state change, not only an entrance?
- Is text supporting the visuals instead of carrying the whole idea?
- Are transitions motivated by the narrative metaphor?
- Are images, icons, and components animated after appearing?
- Can a contact sheet reveal progression without reading every caption?

## Beat-Level Gate

For each beat, mark:

- `main object`
- `start state`
- `end state`
- `motion verb`
- `camera/layer motion`
- `text load`: none / light / heavy
- `failure risk`

If `motion verb` is only fade, slide, or pop, revise.

## Visual Smell List

These make a video feel like a PPT:

- repeated centered title plus three bullets
- large card on plain background with no world around it
- scene cuts that reset all visual context
- decorative icons with no trajectory
- stock-like images that do not move or transform
- every element using the same fade-in easing
- no camera, mask, path, reveal, morph, or system-level motion
- keyframes that look like unrelated posters

## Contact Sheet Review

After snapshots or rendered frames, inspect the grid:

- Is there visible evolution from frame to frame?
- Can the viewer infer the story from motion states?
- Are there repeated static layouts?
- Does any text overflow or dominate the frame?
- Are important objects too small, too dim, or hidden by effects?

If the contact sheet reads like six slide thumbnails, revise before rendering
the final MP4.
